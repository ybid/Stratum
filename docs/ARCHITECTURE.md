## 状态层次

```
TaskState
├── directories: Directory[]
│   └── Directory { id, name, collapsed }
│
├── tasks: Task[]
│   └── Task
│       ├── id
│       ├── directoryId  ──关联 Directory
│       ├── name
│       ├── columns: ColumnDef[]
│       │   └── ColumnDef { id, name, type, options?, width? }
│       └── rows: TaskRow[]
│           └── TaskRow { id, indent, cells, collapsed }
│
├── activeTaskId: string | null
├── sidebarOpen: boolean
├── viewMode: 'outline' | 'flat'
└── locale: 'zh' | 'en'
```

## 视图模式

### Outline 模式 (大纲视图)

```
┌──────────────────────────────────────────────────┐
│ Task A (indent=0)                    ▾ [折叠]   │
├──────────────────────────────────────────────────┤
│   Task A.1 (indent=1)                            │
│     Task A.1.1 (indent=2)                        │
│   Task A.2 (indent=1)                            │
├──────────────────────────────────────────────────┤
│ Task B (indent=0)                                │
└──────────────────────────────────────────────────┘

当 Task A 被折叠时:
- Task A.1, A.1.1, A.2 不显示
- Task A 行显示 ▸ 按钮表示有隐藏子行
```

### Flat 模式 (扁平视图)

所有行都显示，忽略 collapsed 状态，仅显示层级缩进。

## 拖拽交互

### 位置检测算法

```
当拖拽行 A 到行 B 时:

        ▼ before (A 插入到 B 之前)
  ┌─────────┐
  │    B    │
  └─────────┘

        ▲ after (A 插入到 B 之后)

  ┌─────────┐
  │    B    │
  └─────────┘
        ▼ child (A 成为 B 的子级)
  ┌─────────┐
  │    B    │
  ├─────────┤
  │    A    │
  └─────────┘
```

### 层级约束

- `moveRow` 时保留父子关系
- 子行跟随父行移动
- 移动后子行 indent 相对于新父行调整

## 列配置流程

```
用户操作                    组件                    Store
    │                        │                        │
    ▼                        │                        │
右击列头 ──────────────────▶ ColumnConfigModal        │
                              │                        │
                              ▼                        │
                    用户编辑 name/type/options        │
                              │                        │
                              ▼                        │
                    点击保存 ────────────────────────▶ updateColumn()
                                                     │
                                                     ▼
                                              更新 columns[]
                                              调用 persist()
```

## 单元格编辑流程

```
用户操作                    组件                    Store
    │                        │                        │
    ▼                        │                        │
输入内容 ──────────────────▶ CellRenderer            │
                              │                        │
                              ▼                        │
                    onChange 事件 ────────────────▶ updateCell()
                                                     │
                                                     ▼
                                              更新 cells[colId]
                                              调用 persist()
```

## 键盘导航

```
CellRenderer (输入框)
    │
    ├── Enter ──────────────────▶ addRow(当前行后, 同indent)
    │
    ├── Tab ────────────────────▶ updateRowIndent(+1)
    │
    ├── Shift+Tab ─────────────▶ updateRowIndent(-1)
    │
    └── Backspace (空) ────────▶
                                  │
                                  ├── indent > 0 ──▶ updateRowIndent(-1)
                                  │
                                  └── indent = 0 ──▶ removeRow()
```

## 持久化策略

### 写入时机

每个 mutation action 完成后调用 `persist()`

### 读取时机

应用初始化时 (store 首次创建)

### 数据格式

```json
{
  "directories": [...],
  "tasks": [...],
  "activeTaskId": "xxx",
  "sidebarOpen": true,
  "viewMode": "outline",
  "locale": "zh"
}
```

## 组件职责

| 组件 | 职责 |
|------|------|
| `page.tsx` | 组合布局 |
| `Toolbar` | 视图模式切换、语言切换 |
| `Sidebar` | 目录树、任务树、右键菜单 |
| `TaskBoard` | 列头、可见行渲染、拖拽、多选 |
| `ColumnHeader` | 列配置弹窗触发、宽度调整 |
| `ColumnConfigModal` | 列的增删改 |
| `TaskRow` | 单行渲染、缩进、折叠、拖拽 |
| `CellRenderer` | 根据列类型渲染不同输入控件 |
